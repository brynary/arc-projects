## Iteration 1 — Focus: Performance & Stability

- **Cache findNearestSplinePoint per kart per frame**: The most expensive function in the physics loop (`findNearestSplinePoint`) was called 3-4 times per kart per frame (in `isOnRoad`, `getRoadY`, `checkWallCollisions`, and `updatePositions`). Now computed once in `updatePhysics` and cached on `kart._cachedNearest`, with `updateGroundDetection`, `checkWallCollisions`, and `updatePositions` (race.js) all reusing the cached result. Eliminates ~24 redundant spline searches per frame for 8 karts.

- **Particle Color object pooling**: Replaced `new THREE.Color(color)` allocation on every `spawnParticle` call with a reusable `_spawnColor` object. Particle data now stores raw r/g/b floats instead of Color objects, reducing GC pressure during heavy particle emission (drift sparks, boost flames, hit effects).

- **Single-pass wall geometry merge**: Replaced the O(n²) sequential `mergeTwo` approach (which created and copied intermediate Float32Arrays for every wall segment) with a single-pass merge that pre-allocates the final arrays and fills them directly. Eliminates hundreds of intermediate array allocations during track loading.

- **GPU memory leak fix — geometry/material disposal**: Added `disposeObject()` recursive disposal for track groups and kart meshes on restart/quit. Previously, switching tracks or restarting races would leak GPU-side geometry and material buffers. Also fixed a null reference crash in camera.js when `playerKart` is null during state transitions.

- Verified: pass — All 4 tracks load without errors, restart/quit/new-track cycle works cleanly, no console errors

## Iteration 2 — Focus: Driving Feel
- **Fix boost decay (exponential → linear)**: The boost multiplier was self-referencing during decay (`1 + (boostMultiplier - 1) * t`), causing exponential decay instead of linear. Added `boostInitialMultiplier` field stored at boost start, and compute decay as `1 + (initialMultiplier - 1) * t`. Boost now feels stronger and more sustained throughout its duration.
- **Non-linear acceleration curve**: Replaced flat `accel * dt` with a speed-dependent curve: `accel * (1.5 - 0.9 * speedRatio) * dt`. At standstill, acceleration is 1.5x the base rate (snappy start); at top speed, it tapers to 0.6x (satisfying buildup). Low-speed gain is ~11 units/10 frames vs ~6 units at high speed.
- **Drift entry heading snap**: On the first frame of a drift, the kart rotation kicks 0.12 radians (~7°) in the drift direction, giving a crisp "snap" into the drift angle. The `_driftStarted` flag is consumed after one frame so it only fires once.
- **Smoother wall deflection**: Replaced abrupt `atan2` heading assignment with angle-difference blending (70% for glancing, 60% for hard hits). Hard hit speed loss now scales proportionally with impact angle (25-40%) instead of a flat 35%. Stun duration also scales (0.2-0.35s). Walls feel forgiving on grazes and fair on impacts.
- Verified: pass — All 4 tracks load and race cleanly, no console errors. Unit tests confirm linear boost decay, stronger low-speed acceleration, and visible drift snap.

## Iteration 3 — Focus: Track Integrity
- **Y-aware spline lookup for multi-level tracks**: `findNearestSplinePoint` now accepts an optional `kartY` parameter. When provided, Y distance is weighted (0.5× XZ weight) into the distance metric during both the coarse search and ternary refinement. This prevents karts on Volcano Peak's upper switchbacks (Y~60) from snapping to lower spline segments (Y~0) that happen to be nearby in XZ. Updated all callers in physics.js, ai.js, race.js, and track.js utility functions to pass `kart.position.y`.
- **Checkpoint direction validation**: Added travel-direction check to `updateCheckpoints`. The kart's heading (adjusted for reverse speed) is dotted against the checkpoint's forward vector. If the effective dot < -0.2 (i.e., traveling roughly opposite to the checkpoint direction), the checkpoint is rejected. This prevents false lap completions from reversing through checkpoints.
- **Wider wall collision sector search**: Expanded wall sector check from ±1 to ±2 sectors around the kart's current position. On tight tracks with many walls, karts near sector boundaries could previously miss walls one sector away, allowing occasional escape. The wider 5-sector window ensures complete coverage at all speeds.
- Verified: pass — All 4 tracks load and race cleanly with zero console errors. 30-second stress test on Volcano Peak (the most complex multi-level track) confirms stable racing with correct HUD updates and no Y-level teleportation.

## Iteration 4 — Focus: Performance & Stability
- **Cache minimap bounding box & track points**: `updateMinimap()` was calling `centerCurve.getPointAt()` 61 times per frame (at 30 Hz) to recompute the static track outline, plus bounding box min/max and character color hex strings. All of this is now computed once in `buildMinimapCache()` when the track loads, and reused by `updateMinimap()`. Eliminates ~1830 `getPointAt()` calls/second plus string allocations.
- **Swap-and-pop particle removal**: Replaced `activeParticles.splice(i, 1)` (O(n) array shift) with swap-to-end-and-pop (O(1)). During heavy particle emission (8 karts drifting/boosting simultaneously), this avoids shifting up to 200 array elements per removal. Changed iteration from reverse for-loop to forward while-loop to work correctly with the swap pattern.
- **Zero-allocation homing pigeon updates**: Replaced `new THREE.Vector3().subVectors(...)` and `proj.velocity.clone().normalize()` in the homing pigeon steering code with reusable `_toTarget` vector and inline math (manual length computation, divide-by-scalar, and normalize-scale). Eliminates 2 Vector3 allocations per homing pigeon per frame, reducing GC pressure during multi-projectile scenarios.
- **Reuse sort buffer in `updatePositions`**: Replaced `[...allKarts].sort(...)` (which allocates a new 8-element array every physics frame) with a module-level `_sortBuffer` that's filled in-place and sorted. Eliminates ~60 array allocations/second at the fixed timestep rate.
- Verified: pass — All tracks load and race cleanly with zero console errors. Particle system verified under heavy drift/boost load. Minimap renders correctly from cached data.

## Iteration 5 — Focus: Driving Feel
- **High-speed turn rate damping during boost**: When kart speed exceeds normal topSpeed (during boost), turn rate is now reduced proportionally to the overspeed factor (up to 1.6x). At max boost speed, steering is ~36% less responsive. This makes boost feel fast and dramatic without the kart becoming twitchy or uncontrollable at high velocities.
- **Universal overspeed deceleration**: Previously, speed clamping when exceeding effectiveTopSpeed only triggered while the player held accelerate. Now, overspeed decel also applies during coasting and braking (lerp rate 4/s). This eliminates the floaty feel when boost expires while coasting or when driving from road onto offroad without holding throttle.
- **Drift counter-steer modulation**: During drift, steering against the drift direction (counter-steering) now tightens the arc by reducing the drift steer modifier from 0.6 to as low as 0.35, while steering with the drift widens it up to 0.85. This gives skilled players more control over their drift line and makes drifting feel more responsive to input.
- Verified: pass — All 4 tracks (Sunset Bay, Mossy Canyon, Neon Grid, Volcano Peak) load and race cleanly with zero console errors. Drift, boost, coast, and steering all function correctly across all tracks.

## Iteration 6 — Focus: AI Reliability
- **Smart reverse steering**: When an AI kart gets stuck and enters reverse recovery, it now computes the direction toward the nearest spline point (slightly ahead on the track) instead of choosing a random steer direction. This ensures the kart reverses TOWARD the racing line rather than potentially deeper into a wall. Uses `computeReverseSteerBias()` which finds the track center and steers the rear toward it.
- **Faster stuck detection + progressive escalation**: Reduced stuck time threshold from 1.5s to 1.0s for quicker recovery. Added `consecutiveStucks` counter — if the AI gets stuck 3+ times within 8 seconds, reverse duration escalates from 0.8s to 1.5s for a more aggressive recovery maneuver. The counter resets if 8+ seconds pass between stuck events.
- **Post-reverse recovery bias**: After reverse ends, a 0.5s `recoveryBiasTimer` activates shorter look-ahead (50% of normal) and suppresses steering error. This makes the AI home more tightly toward the spline center immediately after unsticking, preventing re-entry into the same wall. The off-road throttle cut is also suppressed during recovery to keep the kart moving.
- **Finished AI karts coast to stop**: Previously, finished AI karts were completely skipped in the update loop, leaving them frozen in place on the track. Now they gradually decelerate (5% per frame) and continue moving forward until stopped, preventing them from being static obstacles blocking the track for other racers.
- Verified: pass — All 4 tracks (Sunset Bay, Mossy Canyon, Neon Grid, Volcano Peak) load and race cleanly with zero console errors. AI karts actively race and overtake the player across all tracks.

## Iteration 7 — Focus: AI Reliability
- **Fix start rotation computation**: The `startRot` for placing karts was computed using the spline tangent at `t≈0`, which on closed CatmullRom curves blends with the return-leg direction and points backward. Replaced with sampling a point 3% ahead on the spline and computing the direction between the nearest point and the ahead point. This fixed all karts starting facing the wrong direction on Neon Grid and Volcano Peak.
- **Pre-corner braking via curvature sampling**: Added `sampleUpcomingCurvature()` that samples 3 points ahead on the AI spline and estimates directional change per unit distance. When curvature > 0.015 and speed > 30, AI coasts or brakes proportionally. This prevents AI from flying off tight 90° corners on Neon Grid at full speed.
- **Smart respawn for karts that never reached a checkpoint**: When emergency respawn or checkpoint stall triggers for a kart with `lastCheckpoint === -1`, the kart is now teleported to the first track checkpoint position instead of its (potentially off-road) start position. This breaks the cycle of respawning to a bad location and getting stuck again.
- **Race-start grace period**: AI karts now accelerate straight ahead for 1.5s at race start with no steering/braking logic. This prevents early-race AI confusion where back-row karts would try to follow the spline and overshoot into walls before even moving.
- **Off-road recovery mode**: When off-road, AI now shortens look-ahead to 60% of base and suppresses steering error to home more directly toward the road center. Combined with existing throttle-on-offroad behavior.
- **Checkpoint stall detection**: Tracks time since last new checkpoint. If 15s pass without a new checkpoint, the kart is respawned. Emergency consecutive-stuck threshold lowered from 5 to 4.
- Verified: pass — Zero console errors on all 4 tracks. Sunset Bay: 8/8 racing. Mossy Canyon: 8/8 racing (smart respawn recovered sprout/pebble). Volcano Peak: 8/8 racing by 15s (previously 5/8). Neon Grid: most karts reach checkpoint 0, remaining recover via stall respawn.

## Iteration 8 — Focus: Collision & Physics
- **Accumulated wall push (anti-jitter)**: Replaced sequential per-wall resolve with a two-phase approach: Phase 1 collects all penetrating walls and accumulates push directions weighted by overlap; Phase 2 applies a single combined push using the normalized accumulated direction and the maximum overlap distance. Heading response uses only the deepest collision. This eliminates oscillation when a kart is wedged in a corner where two walls meet — previously, being pushed out of wall A would push into wall B, causing vibration.
- **Y-level wall collision filtering**: Added vertical extent check to wall collisions. Each wall has `y` (center height) and `height` properties; the filter skips walls whose vertical range [y-0.5, y+height+0.5] doesn't overlap the kart's Y ± 2. On Volcano Peak, karts at Y=18 on upper switchbacks no longer false-collide with walls at Y=0 on lower segments that happen to be nearby in XZ.
- **Kart-to-kart Y-level filtering**: Added `Math.abs(dy) > 4` early-out to `checkKartCollision`. Previously the distance check used full 3D distance (dx²+dz²+dy²) but the push was XZ-only, causing incorrect separation forces between karts on different elevation levels. Now karts on different Y levels (e.g., Volcano Peak switchbacks) are cleanly ignored. Also switched to XZ-only distance for the collision radius check, which is more correct for ground-based karts.
- Verified: pass — All 4 tracks (Sunset Bay, Mossy Canyon, Neon Grid, Volcano Peak) load and race with zero console errors. Volcano Peak confirmed with karts at different Y levels (0.5, 8.9, 18.1) racing without false cross-level collisions.

## Iteration 9 — Focus: Race Flow & State
- **Race epoch guard for stale setTimeouts**: Added a `raceEpoch` counter (in main.js) and `epoch` field (in raceState) that increment on every race start, restart, and quit-to-menu. All 7 setTimeout callbacks (countdown GO hide, results show, startBoostWindow close, lap split fade, final lap banner hide) now capture the epoch at creation time and no-op if the epoch has changed by the time they fire. This prevents stale timeouts from corrupting the new race state — e.g., the results panel popping up during a fresh race after a quick restart.
- **Auto-DNF timeout for AI karts**: After the player finishes, a 30-second `autoFinishTimer` starts counting down. If any AI karts haven't crossed the finish line by then, they're force-marked as `finished` with `finishTime: null` (displayed as "DNF" on the results screen). This prevents the race from hanging indefinitely if an AI kart gets permanently stuck.
- **Complete state reset on restart/quit**: Reset all leaking state between races: `sparkT`/`boostT`/`dustT` particle timers, `prevPlayerLap`, `autoFinishTimer`, and `_prevItem` on all karts (prevents spurious item-pickup sound on the first frame of a new race). Also reset `shieldActive`, `shieldTimer`, and `empLockoutTimer` in `initRace` that were previously missed.
- Verified: pass — Zero console errors. Race epoch increments correctly (1→2 on restart). Stale setTimeouts confirmed harmless after quit-to-menu (results panel stays hidden). All positions unique and race time positive after countdown.

## Iteration 10 — Focus: Performance & Stability
- **Reuse cached spline point in AI steering**: AI `updateSteering` was calling `findNearestSplinePoint` independently with 80 test points at 30Hz per AI kart (~7 × 30 × 96 = 20,160 spline evaluations/sec). Now reuses `kart._cachedNearest` already computed by `updatePhysics` once per kart per frame. Falls back to a fresh computation only if cache is missing. Eliminates the single most expensive redundant computation in the AI system.
- **Pre-allocated AI input objects**: `getAIInput` was creating a new closure object (with 3 captured-variable methods) every physics frame for every AI kart — 420 objects/second of GC pressure. Replaced with a pre-allocated `_AIInput` prototype-based object stored at `kart.ai._input`, whose fields are updated in-place each frame. Zero allocations per frame.
- **Short-circuit `checkStarCollisions`**: The O(n²) star collision loop runs every physics frame. Added a linear scan early-out that skips the nested loop when no kart has `starActive` — which is the common case (stars are rare items). Reduces from 56 iterations to 8 on most frames.
- **Cache minimap canvas 2D context**: `updateMinimap` was calling `document.getElementById` + `getContext('2d')` every update (30Hz). Now cached in `minimapCtx` module variable, invalidated only when `buildHUD` replaces the canvas element.
- **Skip particle update when idle**: `updateParticles` now returns immediately when `activeParticles.length === 0`, avoiding the `instanceMatrix.needsUpdate = true` flag (which triggers a GPU upload) on frames with no visible particles.
- Verified: pass — Zero console errors across all 4 tracks (Sunset Bay, Mossy Canyon, Neon Grid, Volcano Peak). AI input pre-allocation confirmed (`ai._input` present on all AI karts). Cached nearest spline point confirmed (`_cachedNearest` present). 8/8 karts racing on all tracks.

## Iteration 11 — Focus: Crash & Error Fixes
- **Fix physics step cap (3 → 8) + accumulator safety valve**: The game loop capped physics steps at 3 per frame. With FIXED_DT=1/60 and the dt cap of 0.1s, 3 steps only process 0.05s — at 10fps, the game runs at 50% real-time. Increased cap to 8 (covers up to 0.133s dt, well above the 0.1s cap). Added an accumulator safety valve: if it exceeds 0.2s (e.g., tab was backgrounded), it's clamped to FIXED_DT×4 to prevent a burst of catch-up frames. Previously the countdown took forever on slow hardware; now completes reliably.
- **Fix DNF sort order in race results**: When karts have `finishTime: null` (DNF from the auto-finish timeout), `null - number` coerces to 0 in JavaScript, incorrectly placing DNF karts BEFORE timed finishers in the sort. Added explicit null checks in both `updatePositions` and `getRaceResults` sort comparators so DNF karts always sort after all karts with valid finish times.
- **Fix race event priority for player**: The `raceFinished` and `lapCompleted` events were single-value fields that could be overwritten when multiple karts triggered them in the same physics frame. If the player (first in array) finished and then an AI kart also finished in the same step, the AI's event overwrote the player's, and the game never transitioned to RACE_FINISH state. Now events prioritize the player: once the player's event is recorded, AI events won't overwrite it.
- Verified: pass — Countdown completes in ~4s on all 4 tracks (Sunset Bay, Mossy Canyon, Neon Grid, Volcano Peak). DNF karts correctly sort last among finishers. Zero console errors across all tracks and a rapid-restart stress test.

## Iteration 13 — Focus: Camera System
- **Camera wall anti-clip via 2D raycasting**: Added `clipCameraToWalls()` that casts a ray from the kart to the desired camera position and checks for wall segment intersections in nearby sectors (±3 of player sector). When a wall is detected between kart and camera, the camera is pulled to just in front of the wall (95% of intersection distance). Uses Y-level filtering to ignore walls on different elevations. The player's sector index is computed once in `updatePhysics` and stored on `trackData._lastPlayerSector` for the camera to reuse without redundant computation.
- **Speed-dependent chase distance**: Camera offset now scales with kart speed. At standstill, the camera is 13 units behind and 7 units high (tighter, more intimate feel). At top speed, it extends to 18 units behind and 8.5 units high. During boost overspeed (>100% top speed), it pushes to 20 units maximum. This creates a subtle "speed zoom" effect — you feel faster at high speed and get a clearer view of your kart at low speed during countdown and recovery.
- **Camera height floor above road surface**: Uses the kart's `_cachedNearest` spline point (already computed by physics) to ensure the camera never dips below `roadY + 3`. On Volcano Peak's steep downhill plunge and switchbacks, this prevents the camera from going below the road surface when the kart drops in elevation faster than the lerp can follow. The check also applies after wall anti-clip adjustment.
- Verified: pass — Zero console errors on all 4 tracks (Sunset Bay, Mossy Canyon, Neon Grid, Volcano Peak). 8/8 karts racing on every track. Camera sector tracking confirmed working (sector 3 on Sunset Bay after driving).

## Iteration 12 — Focus: Track Integrity
- **Fix road ribbon UV accumulation order**: The texture coordinate distance accumulation (`vDist`) was updated AFTER the UV was pushed to the buffer, causing vertex 0 and vertex 1 to share the same UV v-coordinate (both 0). This produced a zero-width UV band at the start/finish line — the first road segment had no texture stretch (compressed/missing texture). Fixed by moving the distance accumulation BEFORE the UV push so each consecutive sample pair gets a non-zero UV span.
- **Normalize all checkpoint forward vectors across all 4 tracks**: Many checkpoint `forward` vectors were not unit length (e.g., `{0.7, 0, 0.7}` has length 0.9899, `{0.95, 0, 0}` has length 0.95). Since these vectors are used in dot-product direction validation and position-based gating, non-unit vectors produced inconsistent detection behavior at different checkpoints. Normalized all 12 non-unit forward vectors across Sunset Bay (1), Mossy Canyon (3), Neon Grid (1), and Volcano Peak (7) to exact unit length.
- **Scale checkpoint detection forward gate with checkpoint width**: The checkpoint forward-depth gate was hardcoded at 15 units regardless of checkpoint width. For narrow checkpoints (width=10 pier tunnel, width=16 ridge), this 15-unit forward gate was disproportionately large relative to the lateral detection radius, creating an elongated detection zone that could trigger from parallel track segments. Changed to `checkWidth * 0.55`, maintaining a consistent detection aspect ratio: width=28 → gate=15.4, width=22 → gate=12.1, width=16 → gate=8.8, width=10 → gate=5.5.
- Verified: pass — Zero console errors on all 4 tracks (Sunset Bay, Mossy Canyon, Neon Grid, Volcano Peak). All tracks load with correct wall counts and checkpoint counts. Checkpoint normalization confirmed at runtime (0 non-unit vectors). Player progresses through checkpoints correctly on Sunset Bay (reached lap 1 cp 2 within 7s of race time).
